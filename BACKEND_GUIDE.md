# 백엔드 및 데이터베이스 안내 📚

## 현재 블로그 구조 이해하기

### 🎯 지금 사용하는 방식: "파일 기반 블로그"

현재 여러분의 블로그는 **파일 시스템**을 데이터베이스처럼 사용하고 있습니다:

```
blog/
  ├── 회화 MASTER/
  │   ├── 영어/
  │   │   ├── 일일스터디/
  │   │   │   └── 2025-11-28-테스트-글.md  ← 이게 "데이터"입니다!
  │   │   └── 참고자료/
  │   └── 일본어/
  │       ├── 일일스터디/
  │       └── 참고자료/
```

## 현재 시스템의 작동 방식

### ✅ 장점 (중학생도 이해할 수 있는 장점)

1. **설정이 정말 쉬워요!**
   - 데이터베이스 서버 설치 필요 없음
   - 복잡한 설정 파일 없음
   - 그냥 마크다운 파일(.md)만 만들면 됨

2. **Git으로 버전 관리가 쉬워요**
   - 글을 수정하면 Git이 변경사항을 자동으로 추적
   - 언제든지 이전 버전으로 돌아갈 수 있음
   - GitHub에 푸시하면 자동 백업

3. **무료로 배포할 수 있어요**
   - Vercel, Netlify, GitHub Pages 등에서 무료 호스팅
   - 서버 운영 비용 0원!

4. **속도가 빨라요**
   - 빌드 시 모든 페이지를 미리 만들어 놓음 (Static Site Generation)
   - 사용자가 방문하면 이미 만들어진 HTML을 보여줌

### ⚠️ 단점 (현재 시스템의 한계)

1. **글을 작성하면 매번 재빌드가 필요해요**
   ```bash
   npm run build  # 모든 페이지를 다시 만들어야 함
   npm start      # 서버 재시작
   ```
   - 글 1개 추가: 재빌드 필요
   - 글 1개 수정: 재빌드 필요
   - 글 1개 삭제: 재빌드 필요

2. **여러 명이 동시에 글을 쓸 수 없어요**
   - 파일 충돌 문제 발생 가능
   - 누가 무엇을 수정했는지 추적이 어려움

3. **검색 기능이 제한적이에요**
   - 모든 글을 빌드할 때만 검색 가능
   - 실시간 검색 어려움

## 🌟 백엔드와 데이터베이스가 필요한 경우

다음 기능을 원하신다면 백엔드/DB가 필요합니다:

### 📝 필요한 경우:

1. **웹 브라우저에서 바로 글을 작성하고 즉시 게시하고 싶을 때**
   - 재빌드 없이 바로 게시
   - 워드프레스처럼 관리자 화면에서 모든 작업

2. **여러 사람이 함께 블로그를 운영할 때**
   - 각자 계정을 만들어서 로그인
   - 누가 언제 무엇을 작성했는지 기록

3. **댓글 기능을 추가하고 싶을 때**
   - 방문자가 댓글 작성
   - 댓글을 데이터베이스에 저장

4. **좋아요, 조회수, 통계를 보고 싶을 때**
   - 누가 어떤 글을 읽었는지
   - 인기 글 순위

5. **이미지를 블로그에 직접 업로드하고 싶을 때**
   - 드래그 앤 드롭으로 이미지 업로드
   - 자동으로 이미지 최적화

### ❌ 필요 없는 경우 (지금 그대로 좋은 경우):

1. **혼자서 블로그를 운영할 때**
   - 재빌드가 크게 문제되지 않음
   - 파일 구조가 단순하고 명확

2. **글 작성 빈도가 높지 않을 때**
   - 하루에 1-2개 정도 작성

3. **Git과 GitHub에 익숙할 때**
   - 버전 관리가 쉬움
   - 무료 호스팅 활용 가능
